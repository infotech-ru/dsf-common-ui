Утилита для автоматического закрытия всех открытых выпадающих списков `bootstrap-select` при клике на определённые элементы DOM (например, на компонент ComboTree, меню, кнопки и т.п.).

Обработчик устанавливается на фазу захвата (true), что позволяет перехватывать событие до его всплытия и гарантирует закрытие даже в случаях, когда другие обработчики останавливают всплытие.

Если на странице одновременно открыто несколько bootstrap-select, все они будут закрыты.

Функция не проверяет, открыт ли конкретный селект – метод toggle переключит состояние, поэтому открытые закроются, а закрытые откроются.

Для предотвращения закрытия при клике на сам bootstrap-select (например, на его кнопку) необходимо исключить его из контекстов или использовать предикат.

Зависимости

- jQuery (любая версия, совместимая с `bootstrap-select`)

- bootstrap-select (плагин, предоставляющий метод `selectpicker('toggle')`)

{% highlight html %} import { closeOpenSelects, setupCloseSelectOnClick, closeSelectOnComboTreeClick } from './closeOpenSelects'; {% endhighlight %}

API

closeOpenSelects()

Закрывает все открытые в данный момент bootstrap-select на странице.

Параметры: нет

Возвращает: undefined

Используется внутри обработчиков, но может вызываться и явно, например, в колбэках.

setupCloseSelectOnClick(selectors)

Добавляет глобальный обработчик события click (на фазе захвата), который проверяет, произошёл ли клик на одном из указанных контекстов, и если да – закрывает все открытые selectpicker.

Параметры:

selectors (строка | массив строк | функция) – определяет контексты, при клике на которые будет выполняться закрытие:
  • Строка – CSS-селектор, элементы, соответствующие селектору, считаются контекстами.
  • Массив строк – несколько CSS-селекторов.
  • Функция – предикат, принимающий два аргумента: $target (jQuery-объект цели клика) и event (нативное событие). Должен возвращать true, если закрытие необходимо.

Возвращает: undefined

{% highlight html %} // Закрывать при клике на элементы с классами .comboTreeWrapper, .menu-item и .js-edit-tags setupCloseSelectOnClick(['.comboTreeWrapper', '.menu-item', '.js-edit-tags']); // Закрывать при клике на любой элемент внутри #sidebar (кроме самого select) setupCloseSelectOnClick('#sidebar'); // Сложная логика: закрывать, если клик не внутри .keep-open и не на самом select setupCloseSelectOnClick(function($target, event) { return !$target.closest('.keep-open, .bootstrap-select').length; }); {% endhighlight %}

closeSelectOnComboTreeClick()

Готовый обработчик для закрытия при клике на элементы comboTree. Это частный случай setupCloseSelectOnClick, предоставленный для обратной совместимости.

{% highlight html %} [ '.comboTreeWrapper', '.comboTreeDropDownContainer', '.comboTreeInputWrapper', '.comboTreeItem', '.comboTreeItemTitle' ] {% endhighlight %}

Параметры: нет

Возвращает: undefined

Примеры использования

Замена встроенного кода в модальном окне

{% highlight html %} bodySelector.on("click", ".js-edit-tags", function () { $(this).ModalForm("show", { beforeSubmit: function(){ closeOpenSelects(); // вместо старого кода }, }); return false; }); {% endhighlight %}

Закрытие при клике на пункты меню

{% highlight html %} setupCloseSelectOnClick('.navbar-nav .dropdown-item'); {% endhighlight %}

Комбинирование нескольких контекстов

{% highlight html %} setupCloseSelectOnClick([ '.comboTreeWrapper', '.sidebar .menu-item', '#some-button', '.modal-footer .btn' ]); {% endhighlight %}

Использование предиката для динамического определения. Закрывать только при клике вне определённого блока:

{% highlight html %} setupCloseSelectOnClick(function($target) { return $target.closest('.no-close-area').length === 0; }); {% endhighlight %}
