Инструмент на чистом JavaScript для вычисления суммарной высоты дочерних элементов (с учетом их внешних и внутренних отступов) и установки этого значения в качестве CSS-свойства для родительского контейнера. Скрипт поддерживает математические формулы, автоматическое реактивное обновление при изменении DOM или размера окна и интеллектуальный поиск родительских элементов в сложных DOM-структурах.

Возможности

Учитывает реальную отрисованную высоту элементов (getBoundingClientRect), а также их вертикальные margin и padding.

Умный поиск родителя: Сначала ищет родителя как ближайшего предка контейнера (closest). Это позволяет размещать на странице несколько независимых блоков с одинаковыми классами без конфликтов. Если предок не найден, используется глобальный поиск (для обратной совместимости) с выводом предупреждения.

Поддержка кастомных CSS-выражений с плейсхолдерами (например, calc(100vh - {total})), что позволяет создавать сложные адаптивные макеты.

Автоматический пересчет высоты при изменении размера окна (resize) или любых мутациях внутри целевых элементов (MutationObserver).

Все параметры (селекторы целей, родителей, формулы) задаются через data-* атрибуты, что позволяет настраивать логику без изменения JS-кода.

Встроенный механизм отписки от событий и удаления MutationObserver для предотвращения утечек памяти при уничтожении DOM-элементов. Отсутствие зависимостей: Написан на чистом Vanilla JS, не требует jQuery или других библиотек.

API и Инициализация

{% highlight js %} import { ParseHeight, EnableHeightWatch, DisableHeightWatch } from './path/to/parse-height'; // 1. Разовый расчет высоты для всех контейнеров с классом .js-target-set-height ParseHeight('.js-target-set-height'); // 2. Включение автоматического отслеживания изменений (реактивный режим) EnableHeightWatch('.js-target-set-height', { trackResize: true, // Пересчитывать при изменении размера окна trackTargetChanges: true // Пересчитывать при изменении DOM внутри целевых элементов }); // 3. Отключение отслеживания и очистка inline-стилей (например, при закрытии модалки) DisableHeightWatch('.js-target-set-height'); {% endhighlight %}

Data-атрибуты контейнера

Настройка логики расчета осуществляется через data-* атрибуты самого контейнера, на который устанавливается высота.

Реактивный режим (Auto-Update)

{% highlight js %} // Инициализация при открытии модального окна $('#myModal').on('shown.bs.modal', function() { EnableHeightWatch('.modal .js-target-set-height', { trackResize: true, trackTargetChanges: true }); }); // Важный момент: при закрытии модалки нужно очистить observers! $('#myModal').on('hidden.bs.modal', function() { DisableHeightWatch('.modal .js-target-set-height'); }); {% endhighlight %}

Важные замечания и особенности реализации

Скрипт использует алгоритм "сначала ищи рядом". Он вызывает container.closest('.parent-class'). Если предок не найден (например, в старой верстке, где родитель и контейнер находятся в разных частях DOM), скрипт делает глобальный поиск через document.querySelector.

Алгоритм вычислений: Скрипт использует getBoundingClientRect() для получения высоты, что означает, что он считает реальные пиксели на экране, а не значения из CSS. Это гарантирует точность даже при использовании box-sizing: border-box или сложных трансформаций.

Fallback для Padding: Если вы указали data-padding-class, но скрипт не нашел ни одного элемента с этим классом внутри родителя, он автоматически возьмет padding-top и padding-bottom самого родительского элемента (data-parent-class).

Управление памятью: Функция EnableHeightWatch сохраняет ссылки на resize обработчики и MutationObserver в внутреннюю Map. Если вы удаляете контейнер из DOM (например, закрываете SPA-роут или модальное окно), обязательно вызывайте DisableHeightWatch, иначе "висящие" observers будут вызывать пересчеты и потреблять память.

Производительность: MutationObserver настроен на отслеживание изменений атрибутов (style, class), добавления/удаления дочерних узлов (childList: true, subtree: true). При большом количестве DOM-элементов это может вызывать частые пересчеты. Используйте trackTargetChanges: false, если контент статичен.

CSS-свойства: При использовании data-set-type="maxHeight", скрипт корректно преобразует это в element.style.maxHeight. Убедитесь, что в вашем CSS нет правил с !important, которые могут перебить инлайновые стили, устанавливаемые скриптом.
